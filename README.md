# AI Content Factory

A web app for generating complete short-form video content packages from a brand brief. Submit a brief and receive a hook, scripts (15s/30s/60s), shot list, captions, CTA, post caption, and hashtags — all generated by Claude.

## Stack

- **Backend:** Python + FastAPI
- **Frontend:** React + Vite + Tailwind CSS
- **AI:** Anthropic Claude (claude-sonnet-4-6) with prompt caching
- **Database:** SQLite (swap to Postgres via `DATABASE_URL`)

## Quick Start (Docker)

```bash
# 1. Clone and enter the directory
cd ai-content-factory

# 2. Set your API key
cp .env.example .env
# Edit .env and add your Anthropic API key

# 3. Run
docker-compose up --build
```

- Frontend: http://localhost:3000
- Backend API: http://localhost:8000
- Health check: http://localhost:8000/health
- API docs: http://localhost:8000/docs

## Quick Start (Local Dev)

**Backend:**

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
# Set ANTHROPIC_API_KEY in your environment
uvicorn main:app --reload --port 8000
```

**Frontend:**

```bash
cd frontend
npm install
npm run dev
```

## Getting an Anthropic API Key

1. Go to https://console.anthropic.com
2. Sign up or log in
3. Navigate to API Keys
4. Create a new key
5. Copy it into your `.env` file as `ANTHROPIC_API_KEY=sk-ant-...`

## Deploy to Railway

```bash
# Install Railway CLI: https://docs.railway.app/guides/cli
railway login
railway up
```

Set `ANTHROPIC_API_KEY` in Railway's environment variables dashboard.

```bash

fly launch

fly deploy
```

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check |
| POST | `/api/generate` | Generate content from brief |
| POST | `/api/regenerate` | Regenerate a specific section |
| GET | `/api/history` | List all generations |
| GET | `/api/history/:id` | Get a specific generation |
| DELETE | `/api/history/:id` | Delete a generation |
| GET | `/api/metrics` | Get usage metrics and cache stats |

## Prompt Caching

The system prompt (output format + instructions) is marked with `cache_control: {"type": "ephemeral"}` and placed as the first system block. All dynamic brief data goes in the user message. This means the system prompt is cached across requests, reducing input token costs by up to 90% on cache hits.

Cache statistics are logged to the console on every request and available via the `/api/metrics` endpoint.
